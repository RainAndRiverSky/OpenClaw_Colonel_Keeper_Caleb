# Keeper Shared Foundations Exception Register v0.1

## Status

Initial exception register.

This register records approved exceptions, overrides, and compatibility
deviations related to Keeper Caleb's adoption of OpenClaw Shared Foundations.
It does not create exceptions by itself.

## Current Exception State

No Shared Foundations exceptions are currently approved for Keeper Caleb.

## Active Exceptions

None.

## Expired, Retired, Rejected, or Superseded Exceptions

None.

## Known Limitations

The following are current limitations or deferred areas. They are not approved
exceptions and must not be represented as overrides:

- canonical steward identifier registration is pending
- compatibility status tracking is deferred
- requirement-to-evidence mapping is not yet created
- MCP is not enabled
- Obsidian is not enabled
- automated recovery is not implemented
- incident lifecycle automation is not implemented
- severity classification is not implemented
- runtime integration with other stewards is not implemented

## Exception Governance Rule

Any future exception must identify:

- affected Foundation source and version
- affected Keeper Caleb scope
- reason for deviation
- classification and trust-boundary impact
- compatibility effect
- approving authority
- review or expiry condition
- rollback or migration path where applicable

An exception does not inherit to another steward, environment, repository, or
future Foundation version unless explicitly reviewed and approved for that
scope.
